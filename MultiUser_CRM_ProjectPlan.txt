================================================================
   AgriCRM — FULL PROJECT PLAN

----------------------------------------------------------------


----------------------------------------------------------------
USER ROLES
----------------------------------------------------------------
Super Admin  → Everything — all orgs, all users, billing, system health
Org Admin    → Their org only — manage team, settings, data
Sales Rep    → Their own customers 

----------------------------------------------------------------
FEATURE 1: MULTI-TENANT ORGANIZATION SYSTEM
----------------------------------------------------------------
Overview:
  Each company using this app = 1 Organization.

Detail:
  - Super Admin creates Org: "Müller GmbH"
  - Assigns an Org Admin to it
  - Org Admin logs in → sees only their org's data
  - Super Admin can switch between orgs from a dashboard
  - All data (customers, routes, history) is scoped to it

----------------------------------------------------------------
FEATURE 2: USER MANAGEMENT (PER ORG)
----------------------------------------------------------------
Overview:
  Org Admin manages their team — invite, assign roles, deactivate.

Detail:
  - Org Admin goes to Settings → Users
  - Clicks "Invite User" → enters email
  - User gets email link → sets password → lands in that org
  - Org Admin sets role: Rep / Viewer
  - Can deactivate users without deleting their historical data

----------------------------------------------------------------
FEATURE 3: CUSTOMER OWNERSHIP + SHARED POOL
----------------------------------------------------------------
Overview:
  Customers can be assigned to a specific rep or left in a shared
  pool for any rep to claim.

Detail:
  - Rep A imports 50 customers → auto-assigned to him
  - Org Admin creates 20 unassigned customers → shared pool
  - Rep B claims 5 from pool → now assigned to him
  - Org Admin can reassign customers between reps at any time

----------------------------------------------------------------
FEATURE 4: VISIT PLANNING (MULTI-USER AWARE)
----------------------------------------------------------------
Overview:
  Each rep plans their own daily route.
  Org Admin sees a full team calendar.

Detail:
  - Each rep plans their own daily visits
  - Org Admin sees team calendar — who visits which customer, which day
  - If two reps schedule same customer → system warns:
    "Already scheduled by Rep A"
  - Org Admin can lock a customer to prevent scheduling conflicts

----------------------------------------------------------------
FEATURE 5: MAP VIEW (TEAM VIEW)
----------------------------------------------------------------
Overview:
  Color-coded customer map — per rep or org-wide.

Dry Run:
  - Rep sees his own customers on map (green/yellow/red dots)
  - Org Admin sees ALL reps' customers, color-coded per rep
  - Click on customer dot → see assigned rep, last visit, next due
  - Filter by: rep, status, region, date range

----------------------------------------------------------------
FEATURE 6: AUTOMATIC REVISIT SYSTEM
----------------------------------------------------------------
Overview:
  After a visit, system auto-calculates next due date (8 weeks).
  Dashboard shows due visits for each rep.

Detail:
  - Customer visited by Rep A on March 1
  - System marks: due again April 26 (8 weeks later)
  - Rep A dashboard shows: "3 visits due today"
  - Org Admin sees team-wide overdue visits in one panel
  - Overdue customers turn red automatically

----------------------------------------------------------------
FEATURE 7: DAILY ROUTE PLANNER (PER REP)
----------------------------------------------------------------
Overview:
  Start/End location is configurable per user (not hardcoded).
  System suggests optimal route based on due visits.

Dry Run:
  - Rep opens "Plan Today"
  - System suggests customers due today near his start location
  - Rep accepts or removes customers from the route
  - Route calculated → shown on map with optimal order
  - One click → Google Maps opens with all waypoints loaded
  - PDF export of the day's route available

----------------------------------------------------------------
FEATURE 8: SUPER ADMIN DASHBOARD
----------------------------------------------------------------
Overview:
  Global control panel — all orgs, users, system health.

Dry Run:
  - Super Admin logs in → sees all organizations listed
  - Per org stats: active users, total customers, visits this week
  - Can impersonate any Org Admin (for support/debugging)
  - System health panel: DB size, backup status, error logs
  - Billing overview per org (if monetized later)

----------------------------------------------------------------
FEATURE 9: MOBILE MODE (PWA)
----------------------------------------------------------------
Overview:
  React frontend built as a Progressive Web App.
  Fully usable on phone — large buttons.

Dry Run:
  - Rep opens site on phone → mobile layout loads automatically
  - Large tap buttons: Start Visit / Write Note / Open Maps
  - One tap → starts visit timer + opens GPS navigation
  - After visit → quick note entry, auto-saves with timestamp
  - Can be "installed" on home screen like a native app

----------------------------------------------------------------
FEATURE 10: NOTIFICATIONS + REMINDERS
----------------------------------------------------------------
Overview:
  Automated daily reminders for reps and weekly summaries for admins.

Dry Run:
  - Daily 8am → each rep gets email/in-app notification:
    "You have 4 visits due today"
  - If customer overdue 2+ weeks → red badge on customer card
  - Org Admin gets weekly summary: performance + neglected customers
  - Super Admin can configure notification rules per org

----------------------------------------------------------------
FEATURE 11: CSV IMPORT / EXPORT (MULTI-LEVEL)
----------------------------------------------------------------
Overview:
  Import/export at both rep level and org level.

Dry Run:
  - Rep imports his own customer list via CSV
  - Org Admin does bulk import for whole team, assigns reps during import
  - Rep exports his own customer list
  - Org Admin exports entire org's data in one CSV

----------------------------------------------------------------
FEATURE 12: ACTIVITY LOG / AUDIT TRAIL
----------------------------------------------------------------
Overview:
  Every action is logged — who did what, when.

Dry Run:
  - Every action recorded: "Rep B deleted customer XYZ at 14:32"
  - Org Admin can review full activity log for their team
  - Super Admin has access to all org logs
  - Useful for accountability and debugging


----------------------------------------------------------------
BILLING / MONETIZATION MODEL
----------------------------------------------------------------
Billing is charged per Organization (Org Admin pays).
Pricing is based on number of customers stored, billed monthly or yearly.

Suggested Tier Structure:
  Starter  → up to 100 customers  → €9/month  or €89/year
  Growth   → up to 500 customers  → €25/month or €239/year
  Pro      → unlimited customers  → €55/month or €529/year

Rules:
  - Yearly plan gets ~20% discount vs monthly
  - If customer count exceeds current tier → auto prompt to upgrade
  - Super Admin manages all billing, can manually override tiers
  - On payment failure → read-only mode (no new customers, data safe)
  - Free trial: 3 days, up to 20 customers, no card required
